# エンジニア履歴書の書き方テンプレート【通過率UP】

エンジニアの転職において、履歴書・職務経歴書は技術力と経験を効果的にアピールする重要な書類です。書類選考通過率を高める書き方のポイントを詳しく解説します。

## エンジニア特有のアピールポイント

### 技術者として重視される要素
1. **技術スキルの幅と深さ**
2. **問題解決能力**
3. **学習意欲と適応力**
4. **チーム開発経験**
5. **成果・実績の定量化**

### 一般職との違い
- コードやアーキテクチャの具体例
- 技術的な課題解決事例
- パフォーマンス改善の数値
- 使用技術の詳細な記載

## 技術スキルの書き方

### スキルマップの作成

#### プログラミング言語
```
【習熟度の表記例】
★★★ 実務で3年以上、設計から実装まで可能
★★☆ 実務で1-3年、基本的な開発が可能
★☆☆ 基本的な理解、簡単な修正が可能

JavaScript (★★★): React.js, Node.js, TypeScript
Python (★★☆): Django, Flask, データ分析
Java (★☆☆): Spring Boot, 基本的なAPI開発
```

#### フレームワーク・ライブラリ
```
【フロントエンド】
- React.js (3年): Hooks, Context API, Redux
- Vue.js (2年): Vuex, Vue Router, Composition API
- TypeScript (2年): 型定義、ジェネリクス

【バックエンド】
- Node.js (3年): Express.js, Nest.js
- Django (2年): REST API, ORM
- Spring Boot (1年): JPA, Security
```

#### インフラ・ツール
```
【クラウド】
- AWS (2年): EC2, S3, RDS, Lambda
- Docker (2年): コンテナ化、docker-compose
- Kubernetes (6ヶ月): 基本的なデプロイ

【開発ツール】
- Git/GitHub: プルリクエスト、コードレビュー
- CI/CD: GitHub Actions, Jenkins
- 監視: CloudWatch, DataDog
```

### スキル証明の方法
- **GitHub**: ポートフォリオのURL記載
- **資格**: AWS認定、応用情報技術者など
- **貢献**: OSS貢献、技術記事執筆

[GitHubポートフォリオ作成ガイドを見る](/articles/github-portfolio-guide)

## プロジェクト経験の表現方法

### プロジェクト概要の書き方

#### テンプレート
```
【プロジェクト名】: ECサイトリニューアル
【期間】: 2023年4月〜2024年3月（12ヶ月）
【チーム規模】: 6名（PM1名、デザイナー1名、エンジニア4名）
【役割】: フロントエンドリーダー
【技術スタック】: React.js, TypeScript, Next.js, AWS
【概要】: 既存PHPサイトのReact化によるパフォーマンス改善
【成果】: ページ読み込み速度50%改善、コンバージョン率20%向上
```

### 技術的課題と解決策

#### 具体例
```
【課題】: レスポンス速度の遅さ（平均3秒→目標1秒以下）
【原因分析】: 
- N+1問題によるDB負荷
- 画像最適化不足
- バンドルサイズの肥大化

【解決策】:
- SQLクエリの最適化（JOIN活用）
- Next.js Image最適化機能導入
- Code Splittingによる初期読み込み軽量化

【結果】: 平均レスポンス時間0.8秒達成
```

### 定量的な成果の記載

#### 効果的な数値表現
- **パフォーマンス**: 〇〇%高速化、〇〇秒短縮
- **品質**: バグ件数〇〇%削減、テストカバレッジ〇〇%
- **効率**: 開発期間〇〇%短縮、工数〇〇人日削減
- **ビジネス**: 売上〇〇%向上、ユーザー数〇〇倍

## 職種別サンプル

### フロントエンドエンジニア

#### 重点ポイント
- UI/UX改善実績
- パフォーマンス最適化
- レスポンシブデザイン
- アクセシビリティ対応

```
【主な実績】
- React.jsによるSPAの新規開発（MAU 10万人）
- Core Web Vitalsスコア改善（LCP 3.2s→1.1s）
- WAI-ARIA準拠のアクセシブルなコンポーネント設計
- Storybookを活用したデザインシステム構築
```

### バックエンドエンジニア

#### 重点ポイント
- API設計・開発
- データベース設計
- セキュリティ対策
- スケーラビリティ

```
【主な実績】
- REST API設計・開発（100万リクエスト/日処理）
- マイクロサービス化によるモノリス分割
- Redis導入によるDB負荷50%削減
- OAuth2.0 + JWTによる認証基盤構築
```

### インフラエンジニア

#### 重点ポイント
- クラウドアーキテクチャ
- CI/CD構築
- 監視・運用
- コスト最適化

```
【主な実績】
- AWS ECS on Fargateによるコンテナ運用基盤構築
- Terraformによるインフラのコード化
- GitHub ActionsによるCI/CDパイプライン構築
- CloudWatchによる監視・アラート設計（MTTR 50%短縮）
```

## よくある間違いと改善点

### ❌ NG例
```
【悪い例】
- Webサイトを作りました
- バグを修正しました
- チームで開発しました
- 新しい技術を学びました
```

### ✅ OK例
```
【良い例】
- React.jsでSPAのECサイトを開発（月間売上500万円向上）
- SQLインジェクション脆弱性を修正（OWASP Top10対応）
- 4名のスクラムチームでアジャイル開発をリード
- TypeScript導入により型安全性向上、実装ミス30%削減
```

### 改善のポイント
1. **具体的な数値を入れる**
2. **技術名を正確に記載**
3. **ビジネス価値を明確にする**
4. **自分の役割を明確にする**

## 書類選考通過率を上げるコツ

### 企業・職種に合わせたカスタマイズ
- 求人票の必須スキルを強調
- 企業の技術スタックに関連する経験を前面に
- 職種に応じた実績の優先順位調整

### レイアウト・デザイン
- 見やすいフォーマット
- 技術スキル一覧の視覚的整理
- 重要な情報の強調表示

### 定期的な更新
- 新しいプロジェクト経験の追加
- 技術スキルのアップデート
- 資格取得の反映
